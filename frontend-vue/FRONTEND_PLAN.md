# FRONTEND_PLAN.md

## 当前阶段目标

前端当前阶段先构建高质量界面外壳，不接真实后端接口。页面数据全部使用 mock 数据，先保证答辩展示效果、页面结构和业务模块表达清晰。

## 视觉风格

- 简约、干净、大方。
- 低饱和科技蓝为主色。
- 使用卡片化信息组织方式。
- 适合课堂答辩和项目演示。
- 页面应重点体现智慧交通、视觉感知、云边端协同的系统感。

## 页面优先级

### P0

- 系统总览
- 闸机监控
- 拥堵热力图
- 禁停监控
- 道路异常
- 告警中心

### P1

- 设备管理
- 数据统计
- 白名单管理
- 区域配置

### P2

- 模型管理
- 系统设置
- 历史查询增强

## 目录规划

```text
src/components
src/layouts
src/views
src/mock
src/api
```

## Mock 策略

- 后端接口未完成前，页面全部使用 mock 数据。
- mock 数据应尽量贴近后续真实接口的数据结构。
- 后续接入真实接口时，优先只修改 `src/api` 层。
- 页面组件尽量不直接关心数据来自 mock 还是后端。

## 当前不做

- 登录权限
- 复杂用户管理
- 真实后端接口
- 数据库直连
- 算法处理
